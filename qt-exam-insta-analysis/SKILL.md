# 시험지 분석 인스타 게시물 생성 스킬 (qt-exam-insta-analysis)

> **정체성:** "방금 끝난 학교 시험을 **전체 문항 분석표로 펼쳐 보여주고 카드에 넣을 것을 고르게 한 뒤**,
> 디자인은 **A/B/C 샘플에서 고르게 하고**, 정답은 가려진 채 인스타 캐러셀로."

## 이 스킬이 하는 일
학원 강사가 방금 끝난 학교 시험을 분석해 인스타그램에 올리는 카드뉴스 캐러셀을 만든다.
입력은 (1) 시험지 사진 여러 장, (2) 시험범위/교재 텍스트, (3) (선택) 보조자료(본문·어휘표·정답 등).
출력은 `/mnt/user-data/outputs/<게시물명>/` 안의 `card_01.png … card_NN.png` (각 1080×1080).

**핵심 동작 원칙 — 펼쳐 보여주고 고르게 한다.** 이 스킬은 **하네스**다.
1. **전체 문항 선택:** 전 문항을 분석해 **한 화면 분석표**(번호·배점·유형·한 줄 분석·함정도·추천)로
   보여주고, 사용자가 **심층분석할 문항을 고른다**(복수 선택). 나머지 전 문항은 자동으로
   집계(배점표·영역별 특징)에 반영한다. 한 문제씩 30번 캐묻지 않는다.
2. **디자인 선택:** 전체를 만들기 전에 **대표 카드 1장을 A/B/C 세 테마로** 보여주고 고르게 한다.
매 단계의 확정 전에는 다음 단계로 넘어가지 않는다.

## 번들 리소스
- `scripts/crop_question.py` — 문항 크롭 + 모자이크/`--remove-marks`(표기 제거) + 격자 미리보기
- `scripts/build_cards.py` — deck.json → 카드 PNG 렌더(playwright). `--samples`로 A/B/C 샘플 생성
- `assets/base.css` — 공통 스타일(색은 메인색 파생). `assets/theme_a|b|c.css` — 3가지 디자인 테마
- `references/deck-schema.md` — build_cards.py 입력 JSON 스키마(단일 데이터 원천)
- `references/design-spec.md` — 치수/색/폰트 스펙

---

## 🔒 저작권·개인정보 원칙 (반드시 지킴)
- **통째 게시 금지, 부분 모자이크는 허용.** 학교 시험지를 그대로 옮겨 게시하는 것은 안 되지만,
  문항을 캡처한 뒤 **문항의 일부를 모자이크로 가리면** 활용할 수 있다(교육청 답변 근거).
  따라서 deep_dive 에 들어가는 **모든 크롭은 정답/핵심 노출부를 반드시 한 군데 이상 모자이크**한다.
  모자이크가 0개인 크롭은 게시하지 않는다.
- **분석에 필요한 최소 조각만** 자른다. 지문/선지 전체를 길게 노출하지 않는다.
- **저작권 표기(학교명·"무단복제 금지")가 들어간 영역은 크롭 프레이밍에서 잘라낸다**
  (덮어 가리는 게 아니라 박스를 그 위쪽/아래쪽으로 좁혀 애초에 안 들어오게).
- **학생 개인정보**(이름·반·번호·점수)가 페이지에 보이면 그 영역도 `--mask` 로 가린다.

## 전체 흐름 (이 순서를 지킬 것)

### 0단계 — 환경 준비 (말없이)
가장 먼저 렌더링 엔진을 준비한다. 이 과정은 사용자에게 설명하지 않는다.
```bash
export PLAYWRIGHT_BROWSERS_PATH=/opt/pw-browsers
python3 -c "from playwright.sync_api import sync_playwright" || pip install playwright --break-system-packages
python3 -c "import PIL" || pip install pillow --break-system-packages
# 브라우저 바이너리가 /opt/pw-browsers 에 없으면:
#   PLAYWRIGHT_BROWSERS_PATH=/opt/pw-browsers python3 -m playwright install chromium
```
한글 폰트(Noto Sans CJK KR)는 환경에 이미 있다. 없을 경우에만 설치를 안내한다.

### 1단계 — 입력 수집
사용자가 시험지·시험범위를 이미 줬으면 바로 분석으로 넘어간다. 빠진 게 있으면 한 번에 묶어 요청한다:
- 시험지 이미지 (필수): `/mnt/user-data/uploads/` 에서 `view` 로 모두 확인
- 시험범위 텍스트 (필수): 교재명, 과(Lesson), 문법 토픽 목록
- 서술형 정보 (선택): 객관식 답지에는 서술형이 없을 수 있다. 배점표·특징에 서술형이 들어가야
  하면 서술형 문항/배점/정답을 별도로 받는다. 없으면 "서술형 정보 없이 객관식만으로
  분석할까요?" 라고 확인.
- **보조자료 (있으면 반드시 받기 — 심층분석의 깊이를 좌우):** 교재 본문, 어휘 학습지(영영풀이·다의어),
  문법 정리 프린트, 정답·해설, 학원 자체 자료 등. **심층분석은 이 자료의 구체적 근거에서 나온다.**
  없으면 "더 정확한 심층분석을 위해 본문/어휘표/정답 자료가 있으면 주세요"라고 1회 요청한다.

**주의:** 업로드된 이미지 중 **이미 완성된 게시물(레퍼런스)** 과 **분석 대상 시험지** 가 섞여
있을 수 있다. 둘은 다른 시험일 수 있다(예: 레퍼런스는 A중학교, 분석대상은 B중학교). 레퍼런스
게시물은 **포맷/디자인 참고용일 뿐 내용 출처가 아니다.** 어떤 이미지가 분석 대상인지 헷갈리면
반드시 물어 확정한다.

### 2단계 — 전체 문항 분석 & 선택
모든 페이지를 `view`로 읽어 **전 문항을 분석한 뒤, 한 화면 분석표로 펼쳐 보여주고** 사용자가
고르게 한다. 한 문제씩 30번 캐묻지 않는다.

**먼저(말없이):** 시험 메타(학교/학년/시험/일자/총문항·총배점), 전체 유형 분류, 보조자료
(본문·어휘표·문법시트·정답)를 잡아둔다.

**🔍 원본 지문 대비 변형 분석 (원본 지문이 보조자료로 있으면 반드시 — 가장 가치 있는 분석):**
지문·문장 기반 문항(어법·독해·요약·삽입)은 제공된 **교과서 원문과 한 문장씩 대조**해 다음을 찾는다.
- **어디를** 바꿨나 (원문 ↔ 시험 문장)
- **어떻게** 바꿨나 — 시제(과거↔현재완료)·전치사(for↔since)·태·병렬·수일치·to부정사↔동명사/분사·동의어 교체 등
- **실전 체감** — 교과서를 통째로 외운 학생일수록 "똑같네" 하고 방심하다 **바뀐 한 단어**를 놓쳐 틀린다.
학교가 교과서를 어떻게 변형해 어법 오답·함정을 심었는지가 핵심이다. 이 변형 포인트를 분석표의 한 줄
분석과 심층분석 해설에 반드시 반영한다(예: "원문 *came out* → 시험 *have come out*: during+과거시점과
충돌하는 현재완료로 바꿔 오답을 만듦").

**그다음 전체 분석표를 한 번에 제시한다.** 문항마다 한 줄:
| 번호 | 배점 | 유형 | 한 줄 분석(무엇을 어떻게 / 함정) | 함정도(★) | 추천 |
- **추천** 칸: 오답률 높고 변별력 큰 고난도 문항을 **심층분석 후보(★)** 로 표시. 나머지는 집계용.
- 표가 길면 유형별로 묶어 보여줘도 되지만, **전 문항이 한눈에 보이게** 한다.

**표를 보여준 뒤 한 번에 선택받는다(복수 선택):**
- **심층분석 카드로 넣을 문항** — 후보 추천을 참고하되 최종 선택은 사용자. (필요하면 제외할 문항도.)
- 가능하면 입력 UI(체크 형태)로, 안 되면 "9·14·17번" 처럼 번호로 받는다.

**고른 심층분석 문항마다** 제목·해설 초안을 제시해 확인/수정받는다. 해설은 **"학생이 왜·어떻게
틀리는가"**를 1순위로, 보조자료의 **구체적 근거를 인용**해 쓴다(영영풀이 정의 문장, 본문 속 시점
부사 등). "있는 것/없는 것을 구분하면 된다" 식 **동어반복 금지.**

**나머지 전 문항은 자동으로 집계**(배점표·영역별 특징)에 반영한다. 그 뒤 **배점표 / 영역별 특징 /
대비전략 초안**을 구성해 보여준다.
- 영역별 특징·대비전략은 "카드 문구 작성 원칙"을 따른다.
- **대비전략은 이 학교 시험의 패턴에서 도출**(교재·반복 출제 유형·고배점이 몰린 지점). "선지를
  끝까지 봐라" 류 원론적 팁은 **최대 1개**. 나머지는 "현재완료가 본문 어법으로 반복되니 교과서
  본문을 현재완료↔과거로 바꿔 쓰는 연습" 식으로 구체화.

### 3단계 — 브랜딩 & 구성 확정
plan이 모이면 다음을 확인받는다. (가능하면 입력 UI로, 자유 입력 항목은 텍스트로.)

a) **브랜딩** (재사용형이므로 매번 필수):
- 학원명 / 강사명(생략 가능)
- **색 조합(팔레트):** `PALETTES` 프리셋에서 고른다(sunset·navy·ocean·teal·forest·plum·berry·rose·gold·slate).
  `brand.palette`에 이름을 넣는다. 특정 브랜드 색이 있으면 `brand.color`(단색)로 대체 가능.
  **색(팔레트)과 레이아웃(테마)은 독립** — 어떤 색이든 A~I 어느 테마와도 조합된다.
- 표지 제목(학교+학년+과목)·부제 / 해시태그 / 마무리 슬로건

b) **핵심 문법 토픽 확정 (반드시 물어볼 것):** 시험범위에서 뽑은 문법 토픽 초안을 보여주고
   "이대로 갈지 / 뺄지·추가할지 / 강조 순서"를 받는다. 자동으로 넣고 끝내지 않는다.

c) **카드 구성·순서:** 기본 골격 ① 표지 ② 시험범위(문법) ③ 배점표 ④ 영역별 특징
   ⑤~ 심층분석(2단계에서 고른 만큼) → 대비전략 → 마무리. 빼기/순서 변경 허용,
   **표지+마무리 포함 20장 이하**. (심층분석 문항은 2단계에서 이미 확정됨.)
   - **변형 분석 카드(`transform`):** 원본 지문 보조자료가 있으면 강력 추천. 교과서 원문↔시험
     변형을 before→after로 보여준다(카드당 3행 이하). 보통 영역별 특징 뒤 또는 심층분석 앞에 배치.

### 4단계 — 디자인 샘플 (색 팔레트 + 테마 A~I 고르고 보완)
전체를 만들기 전에, 누적 plan으로 **deck.json 초안**을 쓰고 **대표 카드를 여러 조합으로** 렌더해 고르게 한다.

**추천 시작 조합**(무난하게 잘 나오는 것부터 보여줄 것):
- **navy + A** — 파란 블루프린트, 정석·신뢰감
- **teal + B** — 민트 미니멀, 산뜻·깔끔
- **ocean + B** — 블루 미니멀, 가장 깔끔·전문적(무난하게 강력)
- **slate + G** — 크림 뉴스페이퍼, 진중한 "분석 리포트"
- **forest + E** — 연그린 파스텔, 차분·친근(저학년/학부모)
- **rose + E** — 로즈 파스텔, 부드럽고 사랑스러운 톤
- **rose + I** — 로즈 손그림, 캐주얼·친근
- 그 외 고채도 조합(berry+C, ocean+F 등)도 취향에 따라 제시.
- (어두운 테마는 색에 따라 호불호가 커 기본 추천에서는 빼고, 원할 때만 제안.)
```bash
PLAYWRIGHT_BROWSERS_PATH=/opt/pw-browsers \
  python3 scripts/build_cards.py deck.json /mnt/user-data/outputs/<게시물명>/ --samples
# → sample_A.png(폴더탭·블루프린트) / sample_B.png(미니멀 라이트) / sample_C.png(볼드 매거진)
```
- `view`로 3장을 확인시키고 **A/B/C 중 하나를 고르게** 한다.
- "B인데 탭은 A처럼", "색을 더 진하게" 같은 **보완 요청**은 해당 `theme_*.css`(또는 색)를 조정해
  다시 `--samples`로 보여준다. 만족할 때까지 반복.
- 고른 스타일을 deck.json의 `brand.style`(`"A"`/`"B"`/`"C"`)에 적는다. **확정 전 전체 렌더 금지.**

### 5단계 — 문항 크롭 + 표기 제거
심층분석 카드에 들어갈 문항 조각을 시험지 페이지 이미지에서 자동으로 잘라낸다(사용자가 미리
자르지 않음). `scripts/crop_question.py` 사용.

```bash
# (1) 좌표 잡기용 격자 미리보기부터 — view 로 보고 box/mask 비율좌표를 읽는다
python3 scripts/crop_question.py page2.png --grid -o /mnt/user-data/outputs/<게시물명>/crops/_grid2.png

# (2) 문항 박스를 자르고 정답 노출부를 모자이크
python3 scripts/crop_question.py page2.png \
    --box 0.08,0.42,0.94,0.66 \
    --mask 0.70,0.44,0.92,0.50 \
    -o /mnt/user-data/outputs/<게시물명>/crops/q14.png
```
- **가로로 긴 크롭(중요 — 시인성 직결):** 문제 전체·정사각형으로 자르지 말고 **핵심 부분만
  가로가 긴(약 2.5:1~3:1) 형태**로 자른다. 카드의 크롭 슬롯이 가로로 넓고 높이가 제한적이라,
  가로 긴 크롭은 슬롯 폭을 꽉 채워 크고 선명하게 보이지만 **정사각 크롭은 세로에 맞춰 작아진다.**
  예: 어휘 문항이면 지문 전체 대신 "예문 1~2줄 + 선택지"만, 어법이면 밑줄 친 문장 부분만.
- 좌표는 `--grid` 미리보기를 `view` 로 보고 추정 → 1회로 수렴시킨다. 비율(0~1) 기본, 픽셀은 `--px`.
- 출력 PNG 옆에 `q14.json`(원본·box·mask 좌표) 메타가 함께 남아 재현/검수가 된다.
- **모자이크 기본 정책:** ① 빨간펜 정답 표기 ② 함정의 정답이 노출되는 선지 ③ 학생 개인정보.
  "정답이 그대로 보이면 가린다." 마스크 0개로는 게시 금지(스크립트가 경고).

> **채점된 시험지(연필 ○·✓·빨간펜 동그라미)는 `--remove-marks` 를 먼저 써라.**
> 휴대폰으로 찍은 시험지는 원근 왜곡으로 글자 줄이 기울어 있어, 좌표 기반 `--mask` 가
> 한두 줄씩 빗나가 여러 번 재시도하게 된다. `--remove-marks` 는 빨간펜(빨강)과 연필 표기
> (중간톤 회색)를 **색으로** 지우고 검정 인쇄 글자는 남기므로 좌표가 필요 없고 훨씬 안정적이다.
> ```bash
> python3 scripts/crop_question.py page1.jpg --box 0.03,0.175,0.49,0.515 \
>     --remove-marks -o /mnt/user-data/outputs/<게시물명>/crops/q1.png
> ```
> 손글씨가 너무 진해 일부 남으면 `--mask` 로 그 부분만 추가 처리(둘은 함께 쓸 수 있다).
> 단, 이미지는 EXIF 회전이 적용된 **똑바로 선** 페이지여야 한다(찍은 사진은 누워 있을 수 있으니
> `PIL.ImageOps.exif_transpose` 로 세워서 저장한 뒤 크롭).

### 6단계 — 전체 렌더링
확정 데이터를 deck.json(스키마: `references/deck-schema.md`)으로 정리해 넘긴다. 선택한 테마는
`brand.style`에 들어 있어야 한다(없으면 A).
```bash
PLAYWRIGHT_BROWSERS_PATH=/opt/pw-browsers \
  python3 scripts/build_cards.py deck.json /mnt/user-data/outputs/<게시물명>/
```
- `<게시물명>` 은 공백/슬래시/특수문자 없는 단순 이름으로(예: `hangang_m2_final`).
- deep_dive 의 `crop` 경로는 **출력 폴더 기준 상대경로**(예: `crops/q14.png`).

### 7단계 — 검수 & 전달
- **모든 `deep_dive` 카드를 `view` 로 직접 확인** — 정답 노출/모자이크 누락이 한 장이라도
  있으면 사고다. 누락 시 4단계로 돌아가 `--mask` 추가 후 재렌더.
- 나머지 카드도 `view` 로 텍스트 잘림/오타를 점검(최소 표지+배점표+특징 카드).
- `present_files` 로 카드들을 **순서대로** 전달(표지를 첫 번째로).
- 한두 줄 요약만. 긴 후기는 붙이지 않는다.

## 디자인 규칙 요약
세부 치수/색/폰트는 `references/design-spec.md`. CSS는 `assets/base.css`(공통) +
`assets/theme_a|b|c.css`(테마)가 단일 원천이다. 캔버스 1080×1080, `device_scale_factor=2`.
- **6가지 테마** (`brand.style`로 선택, 4단계 `--samples`로 비교) —
  A 폴더탭·블루프린트(격자·흰 외곽선 카드·책더미) / B 미니멀 라이트(밝은 배경, 그림자 카드, 강조 바) /
  C 볼드 매거진(솔리드 배경, 알약 탭, 큰 제목) / D 다크 네온(차콜 배경, 메인색 액센트) /
  E 파스텔 소프트(점 텍스처, 큰 라운드, 불릿 항목) / F 그라디언트(대각선 그라디언트 + 흰 카드).
- 글꼴 `Noto Sans CJK KR`(제목 900, 본문 500/700). 한글은 `word-break: keep-all`.
- **색 = 팔레트, 레이아웃 = 테마, 둘은 독립.** 색은 `brand.palette` 프리셋(`PALETTES`, 10종) 또는
  `brand.color` 단색에서 `resolve_palette()`가 파생한다. 하드코딩 금지. 10색 × 9테마 = 90 조합.
  새 팔레트는 `build_cards.py`의 `PALETTES`에 추가한다.
- **팔레트–테마 궁합:** 고채도 팔레트(sunset·teal·plum·berry·rose·ocean)는 어느 테마와도 잘 맞는다.
  **저채도/차분한 팔레트(slate·navy·gold)**는 검정 잉크가 강한 **뉴스페이퍼(G)**·다크(D·H)에서 스팟감이
  약하니 **밝은 테마(A·B·E·F)** 와 짝짓기를 권한다. 4단계 샘플에서 실제 조합을 확인시키고 고르게 한다.
- 새 카드 타입은 `build_cards.py`에 렌더 함수를 추가하고 `deck-schema.md`에 문서화.
  새 테마는 `theme_*.css`를 추가하고 `build_cards.py`의 `STYLES`에 등록한다.

## 카드 문구 작성 원칙 (가독성 — 매번 지킬 것)
- **완결된 평서문으로 쓴다.** 개조식·명사 종결("~함", "~등 다의어.")은 금지. 자연스럽게 풀어 쓴다.
  - 나쁜 예: `field·match·record 등 다의어.`
  - 좋은 예: `field·match·record 등의 다의어를 여러 뜻으로 활용해 출제했다.`
- **압축하다 의미가 끊기면 안 된다.** 짧게 만들려다 뜻이 안 통하면 한 문장 더 쓰는 게 낫다.
- **애매하게 줄바꿈될 구절은 미리 끊는다.** `'정답 2개·개수 세기'`처럼 한 덩어리로 읽혀야 하는
  표현이 줄 끝에서 쪼개질 것 같으면, 본문 문자열에 `\n`(수동 줄바꿈, 렌더러가 `<br>`로 변환)을
  넣어 위치를 직접 정한다. (한글은 `word-break: keep-all`로 어절 단위 줄바꿈이 기본 적용된다.)
- **심층분석·전략 본문**은 "그래서 학생이 뭘 조심해야 하는지"가 한 번에 와닿게. 원론·동어반복 금지.

## 자주 하는 실수 (피할 것)
- **레퍼런스 게시물 내용을 그대로 베끼기** → 금지. 디자인만 참고, 내용은 분석 대상 시험지에서 뽑는다.
- **분석표 없이 임의로 골라 만들기** → 금지. 2단계에서 전 문항 분석표를 펼쳐 보여주고 사용자가 고른다.
- **디자인을 안 물어보고 바로 전체 렌더** → 금지. 4단계에서 A/B/C 샘플을 먼저 고르게 한다.
- **모자이크/표기 누락** → deep_dive 크롭은 정답·채점표기 제거 + 렌더 후 모든 장 `view` 확인.
- **저작권 표기를 덮어 가리기** → 가리지 말고 크롭 프레이밍에서 잘라낸다.
- **색 하드코딩** → 메인 색상에서 파생.
- **텍스트 넘침** → 본문이 길면 카드에서 잘린다. 렌더 후 확인하고 문구/항목 수를 줄인다.
- **카드 20장 초과** → 캐러셀 한도. 심층분석 장수를 줄인다.
- **개조식 문체("~함")** → 금지. 완결된 평서문으로 푼다(위 "카드 문구 작성 원칙").
- **당연한 해설** → "있는 것/없는 것을 구분하면 된다" 류 동어반복 금지. 구체적 함정·오답 이유를 짚는다.
- **원론적 대비전략** → 학교 시험 특징에서 도출. 일반 팁은 1개 이하.
- **문법 토픽 무단 확정** → scope 카드 문법은 사용자에게 확인받고 넣는다.
