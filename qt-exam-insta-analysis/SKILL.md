# 시험 분석 콘텐츠 생성 (qt-exam-insta-analysis)

> **정체성:** "방금 끝난 학교 시험을 해부해서 — 홍보용이면 **인스타 카드뉴스**로,
> 내부/배포용이면 **A4 상세 분석지**로. 정답은 가려진 채, 디자인은 고르게 하고."

## 이 스킬이 하는 일
학원 강사가 방금 끝난 학교 시험을 분석해 콘텐츠를 만든다. 산출물은 두 모드:

| 모드 | 산출물 | 성격 |
|--|--|--|
| **📱 인스타 카드뉴스** | `card_01~NN.png` (1080×1080) | 홍보용 하이라이트 — 표지·배점표·특징·심층분석·전략 카드 |
| **📄 A4 상세 분석지** | `report.pdf` (여러 페이지) | 전수 분석 내부/배포 자료 — 총평·**전 문항 분석표**·변형 전수·상세 해설 |

입력(공통): (1) 시험지 사진/PDF (2) 시험범위 텍스트 (3) 보조자료 — 교과서 본문·어휘표(영영풀이)·
문법 정리·정답. 보조자료는 카드뉴스에선 권장, **상세 분석지에선 사실상 필수**(변형 분석의 근거).

## 번들 리소스
- `scripts/crop_question.py` — 문항 크롭 + 모자이크/`--remove-marks`(채점 표기 제거) + 격자 미리보기 (공통)
- `scripts/build_cards.py` — deck.json → 카드 PNG(playwright). `--samples`로 디자인 샘플 생성 (카드뉴스)
- `scripts/build_report.py` — report.json → A4 PDF(playwright). HTML도 함께 저장 (분석지)
- `assets/base.css` + `theme_a~i.css` — 카드 9종 테마 / `assets/report.css` — A4 인쇄 스타일
- `references/deck-schema.md` / `report-schema.md` — 각 렌더러 입력 스키마(단일 데이터 원천)
- `references/design-spec.md` — 카드 치수/색/폰트 스펙

## 🔒 저작권·개인정보 원칙 (두 모드 공통 · 반드시 지킴)
- **통째 게시 금지, 부분 가림은 허용.** 문항을 캡처해 쓸 때는 정답·채점 표기 등 **일부를 반드시
  가린다**(교육청 답변 근거). 채점된 시험지는 `--remove-marks`(빨간펜·연필 표기 색 기반 제거)가 기본.
- **분석에 필요한 최소 조각만** 자른다. 저작권 표기(학교명·"무단복제 금지") 영역은 크롭 프레이밍에서
  잘라낸다(덮지 말 것). **학생 이름·점수 등 개인정보**는 크롭에서 배제하거나 `--mask`로 가린다.
- 학생이 쓴 서술형 답안(=모범답 노출)도 가린다. 리포트여도 외부 배포 가능성을 전제로 같은 기준.

## 전체 흐름

### 0단계 — 출력 형태 확정 (가장 먼저 물을 것)
사용자가 이미 명시했으면(예: "카드뉴스 만들어줘", "상세 분석지로") 묻지 않고 해당 모드로 간다.
불명확하면 **첫 질문으로** 확인한다:
- **📄 A4 상세 분석지** — 전 문항 전수 분석, 인쇄/배포용 PDF
- **📱 인스타 카드뉴스** — 홍보용 하이라이트 카드 세트
- **둘 다** — 분석(2단계)은 한 번만 하고 두 산출물을 모두 만든다(분석지 먼저, 카드뉴스가 그 요약).

### 0.5단계 — 환경 준비 (말없이)
```bash
export PLAYWRIGHT_BROWSERS_PATH=/opt/pw-browsers
python3 -c "from playwright.sync_api import sync_playwright" || pip install playwright --break-system-packages
python3 -c "import PIL" || pip install pillow --break-system-packages
python3 -c "import numpy" || pip install numpy --break-system-packages
# 브라우저가 없으면: PLAYWRIGHT_BROWSERS_PATH=/opt/pw-browsers python3 -m playwright install chromium
```
한글 폰트(Noto Sans CJK KR)는 환경에 이미 있다고 가정한다.

### 1단계 — 입력 수집 (공통)
- **시험지 (필수):** 전 페이지를 `view`로 확인. 휴대폰 사진은 EXIF 회전이 있을 수 있으니
  `PIL.ImageOps.exif_transpose`로 세워 저장한 뒤 크롭에 쓴다. **PDF로 오면** `pdftoppm -png -r 200`
  (또는 pymupdf)으로 페이지를 이미지로 뽑는다 — 결과가 너무 크면(8천px+) 3000px 폭으로 줄여 작업.
- **시험범위 텍스트 (필수).**
- **보조자료:** 교과서 본문·어휘표·문법시트·정답. 있으면 **교과서 원문 대비 변형 분석**이 가능해진다.
  없으면 1회 요청하고, 그래도 없으면 분석지 총평에 그 한계를 명시한다.
- 레퍼런스(완성 예시)와 분석 대상이 섞여 있으면 반드시 물어 확정한다.

### 2단계 — 전수 분석 (공통 · 스킬의 핵심)
모든 페이지를 읽고 **전 문항**을 분석해 **한 화면 분석표**로 펼쳐 보여준다(한 문제씩 캐묻지 않음).

1. **시험 메타:** 학교/학년/시험/일자/총문항/총배점/객관식·서술형 구성.
2. **문항 전수 분석표:** 문항마다 — 배점 / 유형 / **출제 근거**(교과서 몇 과 본문·어휘표·문법.
   자료 없으면 지문·영역 기준) / 난이도(★0~3) / 한 줄 분석(무엇을 어떻게 물었고 함정이 뭔지).
3. **🔍 변형 분석(원본 자료가 있을 때 — 가장 가치 있는 분석):** 지문·문장 기반 문항을 원문과
   한 문장씩 대조해 **찾은 변형을 전부** 기록 — 어디를(원문↔시험), 어떻게(시제·전치사·수일치·병렬·
   to부정사↔동명사·동의어 교체 등), 실전 체감(외운 학생일수록 "똑같네" 하고 방심하다 바뀐 한 단어를
   놓친다).
4. **심층/상세 해설 후보:** 오답률 높고 변별력 큰 고난도 문항(카드뉴스 2~5개 / 분석지 4~8개).
   해설은 **"학생이 왜·어떻게 틀리는가"** 1순위, 보조자료의 **구체적 근거 인용** 필수.
   "있는 것/없는 것을 구분하면 된다" 식 동어반복·당연한 해설 금지.
5. **대비전략:** 이 학교 시험의 패턴에서 도출(반복 유형·고배점 몰린 지점). 원론적 팁 최대 1개.

분석표를 보여준 뒤 **심층/상세 해설에 넣을 문항을 한 번에 선택**받는다(복수 선택, 추천 표시).

### 3단계 — 확정 대화 (모드별)
**공통:** 심층/상세 문항 확정 + 문구 수정. **확정 전에는 렌더하지 않는다.**

**📱 카드뉴스 모드:**
- **브랜딩(매번 필수):** 학원명/강사명(생략 가능), **색 팔레트**(`PALETTES` 10종: sunset·navy·ocean·
  teal·forest·plum·berry·rose·gold·slate — 또는 `brand.color` 단색), 표지 제목·부제/해시태그/슬로건.
- **핵심 문법 토픽 확정(반드시 물을 것):** scope 카드에 들어갈 문법 초안을 보여주고 가감·순서를 받는다.
- **카드 구성:** 기본 골격 ①표지 ②시험범위 ③배점표 ④영역별 특징 (⑤변형 분석 — 원본 자료 있으면
  강력 추천, `transform` 카드·행 3개 이하) ⑥~심층분석 →대비전략 →마무리. **표지+마무리 포함 20장 이하.**

**📄 분석지 모드:**
- **브랜딩:** 학원명/강사명(선택), 악센트 색 1색(기본 네이비 `#1E2A78`) — 인쇄물이라 색은 1색만.
- **섹션 구성:** 기본 순서(총평→배점→전수표→변형→상세해설→매핑→전략)에서 빼기/순서 변경 허용.

### 4단계 — 디자인 샘플 (카드뉴스 모드만)
전체 렌더 전에 **대표 카드를 여러 조합으로** 렌더해 고르게 한다:
```bash
python3 scripts/build_cards.py deck.json <출력폴더>/ --samples   # 테마별 샘플 생성
```
**추천 시작 조합**(무난하게 잘 나오는 것부터): navy+A(블루프린트·정석) / teal+B(미니멀·산뜻) /
ocean+B(가장 깔끔·전문적) / slate+G(뉴스페이퍼·리포트 톤) / forest+E(파스텔·친근) /
rose+E(파스텔·사랑스러움) / rose+I(손그림·캐주얼). 그 외 고채도 조합(berry+C, ocean+F 등)도 취향에
따라. 어두운 테마(D·H)는 호불호가 커 기본 추천에서 제외, 원할 때만. 보완 요청은 `theme_*.css`/색을
조정해 재샘플. 확정 값은 deck.json의 `brand.palette`(또는 `color`)와 `brand.style`(A~I)에.
(분석지 모드는 단일 인쇄 디자인이라 이 단계 없음 — 악센트 색만 3단계에서 확정.)

### 5단계 — 문항 크롭 (공통)
```bash
# 좌표는 격자 미리보기로 확인
python3 scripts/crop_question.py page1.png --grid -o crops/_grid1.png
# 채점된 시험지는 --remove-marks 기본, 필요시 --mask 병용
python3 scripts/crop_question.py page1.png --box 0.05,0.36,0.49,0.50 --remove-marks -o crops/q8.png
```
- **가로로 긴(2:1~3:1) 크롭이 기본** — 핵심부(발문+선지, 밑줄 문장)만. 정사각/문항 전체 크롭은
  슬롯에서 작아진다. 분석지는 세로 여유가 있어 전체 크롭도 허용(높이 62mm 제한).
- 진한 연필/펜 필기가 `--remove-marks`로 안 지워지면 **PIL로 흰 사각형을 덧그려** 정리한다
  (픽셀화 마스크는 remove-marks와 겹치면 지저분해짐). 학생 서술형 답안 줄은 통째로 흰칠.
- 잘린 조각은 반드시 `view`로 정답·개인정보 노출 점검.

### 6단계 — 렌더링
**📱 카드뉴스:** deck.json(스키마: `references/deck-schema.md`) →
```bash
python3 scripts/build_cards.py deck.json <출력폴더>/
```
**📄 분석지:** report.json(스키마: `references/report-schema.md`) →
```bash
python3 scripts/build_report.py report.json <출력폴더>/report.pdf
```
- crop 경로는 출력 폴더 기준 상대경로(예: `crops/q8.png`). 분석지의 쪽나눔은 섹션에 `"page_break": true`.

### 7단계 — 검수 & 전달 (공통)
- **크롭이 들어간 카드/페이지 전부**를 `view`로 확인 — 정답·개인정보 노출이 한 장이라도 있으면 사고.
- 카드뉴스: 텍스트 잘림/오타 점검(최소 표지+배점표+특징). 분석지: PDF 전 페이지 훑기(표 잘림·쪽나눔).
- 파일 전달 + 한두 줄 요약. 긴 후기는 붙이지 않는다.

## 디자인 규칙 요약 (카드뉴스)
세부는 `references/design-spec.md`. 캔버스 1080×1080, `device_scale_factor=2`.
- **색 = 팔레트, 레이아웃 = 테마, 둘은 독립.** `resolve_palette()`가 파생, 하드코딩 금지. 10색×9테마.
- **팔레트–테마 궁합:** 고채도는 어디든. 저채도(slate·navy·gold)는 잉크 강한 G·다크(D·H)에서 스팟감이
  약하니 밝은 테마(A·B·E·F)와 짝을 권장.
- 테마 9종: A 블루프린트 / B 미니멀 / C 볼드매거진 / D 다크네온 / E 파스텔 / F 그라디언트 /
  G 뉴스페이퍼 / H 네온사인 / I 손그림. 페이지 번호(1/N)·워터마크 자동.
- 새 카드 타입은 `build_cards.py`+`deck-schema.md`, 새 테마는 `theme_*.css`+`STYLES` 등록.

## 문구 작성 원칙 (두 모드 공통)
- **완결된 평서문.** 개조식("~함")·명사 종결 금지. 전수표의 "한 줄 분석"은 간결하되 뜻이 끊기지 않게.
- **구체 근거 인용.** "어휘표의 영영풀이 `to think that…`이 그대로 선지로 나왔다"처럼 자료를 직접
  가리킨다. 동어반복·당연한 말 금지.
- **전략은 학교 맞춤.** 반복 패턴·고배점 몰린 지점에서 도출. 원론 팁 최대 1개.
- 한 덩어리로 읽혀야 하는 구절이 줄 끝에서 쪼개질 것 같으면 `\n` 수동 줄바꿈(렌더러가 `<br>` 변환).

## 자주 하는 실수 (피할 것)
- **출력 형태를 안 묻고 시작** → 금지. 0단계에서 카드뉴스/분석지(/둘 다)를 확정한다.
- **분석표 없이 임의로 골라 만들기** → 금지. 전 문항 분석표를 펼쳐 보여주고 사용자가 고른다.
- **분석지에서 일부 문항만 표에 넣기** → 금지. 전수표는 분석지의 존재 이유다.
- **디자인을 안 물어보고 전체 렌더**(카드뉴스) → 금지. 4단계 샘플 선택 먼저.
- **모자이크/표기 제거·개인정보 처리 누락** → 크롭 전부 `view` 검수.
